# 分类不能当回归做

## 核心矛盾

```
回归的数值标签 → 隐含大小顺序 → 模型学到"1和2相近"
分类的真实含义 → 类别独立无顺序 → 猫≠狗≠鸟，三者等距
```

## 错误做法 vs 正确做法

| | 回归思路（❌） | 分类思路（✅） |
|---|---|---|
| 输出层 | 1 个神经元 | N 个神经元（N=类别数） |
| 输出值 | 一个连续数值 | 概率分布（Softmax） |
| 损失函数 | MSELoss | CrossEntropyLoss |
| 标签 | 1, 2, 3（数值） | 0, 1, 2（索引）或 one-hot |
| 学到什么 | 数字大小关系 | 每个类别的置信度 |

## 直观理解

给类别编号 1=猫、2=狗、3=鸟：

- 模型输出 1.8，离狗(2)很近 → MSELoss 很小 → 模型觉得"快对了"
- 但 1.8 既不是猫也不是狗，是分类错误

CrossEntropy 只看"真实类别的概率够不够高"，不关心数字距离。

## 代码对比

```python
# ❌ 错误：回归方式做分类
model = nn.Linear(784, 1)        # 输出1个数值
loss_fn = nn.MSELoss()           # MSE
labels = torch.tensor([1.0, 2.0, 3.0])  # 猫=1, 狗=2, 鸟=3

# ✅ 正确：分类方式
model = nn.Linear(784, 3)        # 输出3个值
loss_fn = nn.CrossEntropyLoss()  # 内置 Softmax
labels = torch.tensor([0, 1, 2]) # 0/1/2 只是索引编号，不是数值
```

## 一句话

> 分类用 CrossEntropyLoss + Softmax，不是因为更复杂，而是因为它不引入数字之间的大小假设。
