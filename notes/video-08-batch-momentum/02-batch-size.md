# Batch Size 的影响

**优先级：⭐ 重要**

## 一句话

> **Batch size 影响训练的稳定性、速度和模型泛化能力，太大或太小都有问题。**

## Batch Size 对比

| batch size | 特点 | 问题 |
|---|---|---|
| **很小** (1~4) | 每次更新快，噪声大 | 梯度估计不准，训练震荡 |
| **适中** (32~256) | 常用范围，效果好 | — |
| **很大** (512~) | 梯度稳定，一次更新慢 | 容易收敛到"尖锐"的最低点，泛化差 |

## 为什么大 batch 有时反而不好？

小 batch 的噪声帮助模型跳出局部最小值：

```
大 batch (梯度精确)：
  精确指向最低点 → 容易掉进最近的局部最小值（尖锐的底）
  
小 batch (梯度有噪声)：
  每次方向有随机偏移 → 有机会跳出小坑 → 找到更平坦的谷底
```

**平坦的谷底泛化更好**——测试数据稍有偏移，loss 变化不大。

## 实际建议

| 场景 | 推荐 batch size |
|---|---|
| 一般分类任务 | 32 ~ 128 |
| 大模型训练（显存有限） | 1 ~ 16（配合梯度累积） |
| 图像任务 | 64 ~ 256 |
