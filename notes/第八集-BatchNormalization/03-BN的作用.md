# Batch Normalization 的作用

**优先级：📌 了解即可**

## 一句话

> **BN 让 error surface 变得更平滑，梯度下降更容易找到最低点。**

## BN 的两个可能解释

### 解释一：Internal Covariate Shift（内部协变量偏移）

**早期的观点：**

> 每层参数更新后，上一层输出的分布变了，下一层要不断适应新的分布。

BN 让每层输出的统计量（均值、方差）保持稳定，下一层面对的是"稳定"的输入。

### 解释二：改变 Error Surface 的形状（现在更被认可）

**实验发现（2018 年论文）：**

> BN 的主要作用不是解决 Internal Covariate Shift，而是**改变了 error surface 的地形**。

```
没有 BN：
  error surface 崎岖不平
  梯度方向变化剧烈
  需要很小的学习率

有 BN：
  error surface 更平滑
  梯度方向更稳定
  可以用更大的学习率
```

## BN 的副作用（理解即可）

| 特点 | 说明 |
|---|---|
| 训练更快 | 梯度更稳定，可以用更大的 lr |
| 对初始化不敏感 | 即使初始化不好，BN 也能兜底 |
| 有正则化效果 | 因为每个 batch 的 μ、σ 都是估计值，引入了随机噪声 |
| **依赖 batch size** | batch 太小 → μ、σ 估计不准 → BN 效果差 |

## 大模型为什么不用 BN

DeepSeek / GPT / LLaMA 都用 **Layer Normalization**，原因：

```
BN：对同一个 batch 的不同样本在同一层的"同一通道"做归一化
    依赖 batch size
    训练和推理行为不同

Layer Norm：对同一个样本的"所有特征"做归一化
            不依赖 batch size
            训练和推理行为一致
```
