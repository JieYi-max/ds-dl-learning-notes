# Batch Normalization（批量归一化）

**优先级：⭐ 重要**

## 一句话

> **把 Feature Normalization 应用到神经网络每一层的输出上，但基于一个 batch 的统计量来算 μ 和 σ。**

## 为什么不在输入层做一次就够了？

神经网络中，**每一层的输出会成为下一层的输入**：

```
输入 x → 第1层 → z₁ → 激活函数 → a₁ → 第2层 → z₂ → ...
```

即使输入 x 归一化了，经过第一层权重 W₁ 的计算后，z₁ 的分布又会变得"不规整"。所以**每一层都需要归一化**。

## BN 的计算流程

对于一个 batch 的样本，在某一层的输出 z 上：

```
Step 1: 算均值 μ = (z₁ + z₂ + ... + z_B) / B
Step 2: 算标准差 σ = sqrt( Σ(z_i - μ)² / B )
Step 3: 归一化  ż_i = (z_i - μ) / σ
Step 4: 恢复表达能力  ẑ_i = γ · ż_i + β
```

## 为什么要有 γ 和 β？

归一化后数据均值=0、标准差=1，但网络**可能不需要数据严格在 0 附近**。

```
γ（缩放参数）：控制归一化后的数据范围
β（偏移参数）：控制归一化后的数据偏移

网络自己学习 γ 和 β 的最优值

极端情况：γ = σ, β = μ  →  BN 退化为恒等映射，什么都不做
```

## 训练 vs 推理时的区别

| | 训练时 | 推理时 |
|---|---|---|
| μ、σ 怎么算 | 用当前 batch 算 | 用训练时累积的全局均值/方差 |
| 为什么 | batch 不同 μ、σ 不同，增加随机性 | 推理时没有 batch，用固定的统计量 |
