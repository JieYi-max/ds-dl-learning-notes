# BN 为什么有效 & 局限

**优先级：⭐⭐ 重要（理解层面）**

**对应课件：** `normalization_v4.pdf` 第 10-12 页

---

## 一句话

> **BN 让 error surface 变平滑，梯度更稳定，训练更快。但它依赖 batch size，大模型改用 Layer Normalization。**

---

## BN 为什么有效（PPT 第 10-11 页）

### 解释一：Internal Covariate Shift（传统观点）

> 每层参数更新后，上一层输出的分布就变了，下一层要不断适应新的输入分布。

BN 让每层输出统计量（均值、方差）保持稳定，下一层面对的输入分布更稳定。

### 解释二：平滑 Error Surface（当前研究认可的观点）

2018 年的研究发现，上面的解释不对：

> **BN 的真正作用不是解决 Internal Covariate Shift，而是改变了 error surface 的地形。**

```
没有 BN：error surface 崎岖不平，梯度方向变化剧烈
   → 需要小学习率，训练不稳定

有 BN：error surface 变平滑，梯度方向更稳定
   → 可以用更大的学习率，训练更快
```

---

## BN 的三个好处（PPT 第 11 页）

| 好处 | 原因 |
|---|---|
| 训练更快 | 梯度更稳定，可以用更大的学习率 |
| 对初始化不敏感 | 即使初始化不好，BN 会把输出拉回合理范围 |
| 有轻微正则化效果 | 每个 batch 的 μ、σ 是估计值，带有随机噪声 |

---

## BN 的局限（PPT 第 12 页）

### 局限 1：依赖 batch size

```
batch size 很大（≥32）：μ、σ 估计准确 → BN 效果好 ✓
batch size 很小（≤8）： μ、σ 波动大 → BN 效果差 ✗
batch size = 1：          μ 就是自己 → BN 完全失效 💀
```

### 局限 2：训练和推理行为不同

训练时用 batch 统计量，推理时用全局统计量——模型在两种模式下运行，给部署带来额外复杂度。

---

## 替代方案：Layer Normalization（大模型的选择）

DeepSeek / GPT / LLaMA **全部用 Layer Normalization，不用 BN**：

| | Batch Normalization | Layer Normalization |
|---|---|---|
| **对什么归一化** | 对 batch 维度（同通道不同样本） | 对特征维度（同样本不同特征） |
| **公式** | ẑ = γ·(z-μB)/σB + β | ẑ = γ·(z-μfeat)/σfeat + β |
| **依赖 batch size** | ❌ 依赖（需多个样本算 μ、σ） | ✅ 不依赖（单个样本就能算） |
| **训练=推理** | ❌ 不一致（推理用全局统计量） | ✅ 一致（推理和训练计算一样） |
| **适用场景** | CNN（图像任务） | Transformer（大模型/序列任务） |

```
为什么大模型用 LN 不用 BN？

训练大模型时 batch size 受显存限制很小（1~16）
BN 在小 batch 下效果差
LN 不依赖 batch size，且训练推理一致
```

---

## 其他归一化变种（PPT 第 12 页）

| 方法 | 归一化方向 | 适用场景 |
|---|---|---|
| **BatchNorm** | 对 batch 归一化 | CNN，大 batch |
| **LayerNorm** | 对特征归一化 | Transformer/RNN |
| **InstanceNorm** | 对单样本单通道归一化 | 图像风格迁移 |
| **GroupNorm** | 将通道分组归一化 | 小 batch 视觉任务 |
