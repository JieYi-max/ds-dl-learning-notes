# 为什么需要归一化

**优先级：⭐⭐⭐ 重要**
**课件：** `normalization_v4.pdf` 第 2-4 页

---

💡 **一句话：不同特征的数值范围差异导致 loss 地形崎岖不平，归一化让地形变平滑，梯度下降更容易走。**

---

## 问题：输入范围不同 → 梯度大小不同

假设 y = w1·x1 + w2·x2 + b

x1 范围小（0~1），x2 范围大（100~200）

![梯度公式](./梯度公式.png)

梯度大小跟输入 x 直接相关：

```
x1 小（0~1）     → ∂L/∂w1 小     → w1 更新慢     → w1 方向地形平坦
x2 大（100~200） → ∂L/∂w2 大     → w2 一步飞了   → w2 方向地形陡峭

loss 地形：在 w1 方向被压扁，w2 方向被拉长
梯度下降走 Z 字形，训练困难
```

---

## 解决方案：Feature Normalization

对每个特征维度做归一化：

![特征归一化公式](./特征归一化公式.png)

其中：

![均值公式](./均值公式.png)
![标准差公式](./标准差公式.png)

符号说明：
- **x_i^r** — 第 r 个样本的第 i 个特征（如第 3 套房面积 200）
- **mi** — 所有样本在特征 i 上的均值（(120+80+200)/3 ≈ 133）
- **σi** — 所有样本在特征 i 上的标准差（≈ 49）

归一化效果：
- 每个特征均值=0，标准差=1
- 所有特征尺度一致
- 梯度更新直指最低点，训练更快

---

## 代码示例

```python
import torch

x = torch.tensor([[120.0, 3.0], [80.0, 2.0], [200.0, 5.0]])

mean = x.mean(dim=0)
std = x.std(dim=0)
x_norm = (x - mean) / std

print(x_norm)
```

🔗 **关联知识：**
- 跟 RMSProp/Adam 解决的是同一个问题，但 Normalization 在数据层面改，RMSProp 在优化器层面改
- 两者配合使用效果最好
