# 为什么需要归一化

**优先级：⭐⭐⭐ 重要**
**课件：** `normalization_v4.pdf` 第 2-4 页
---

💡 **一句话：不同特征的数值范围差异导致 loss 地形崎岖不平，归一化让地形变平滑，梯度下降更容易走。**
---
## 问题：输入范围不同 → 梯度大小不同
假设 y = w1·x1 + w2·x2 + b
x1 范围小（0~1），x2 范围大（100~200）
为什么 x 的范围会影响梯度？用最简单的模型算一遍：
```
用 MSE loss：L = (y_pred - y_true)²
对 w1 的梯度：∂L/∂w1 = 2 × (y_pred - y_true) × x1
对 w2 的梯度：∂L/∂w2 = 2 × (y_pred - y_true) × x2
                ↑                  ↑
           误差项相同          输入值不同
```
关键：梯度 = 误差 × 输入值
```
x1 小（0~1）     → ∂L/∂w1 小     → w1 更新慢     → w1 方向地形平坦
x2 大（100~200） → ∂L/∂w2 大     → w2 一步迈很远 → w2 方向易震荡
合起来：loss 地形在 w1 方向被压扁，w2 方向被拉长
梯度下降走 Z 字形，训练困难
```
---
## 解决方案：Feature Normalization
对每个特征维度做归一化：

其中：

![均值公式](./均值公式.png)
![标准差公式](./标准差公式.png)

符号说明：
- **x_i^r** — 第 r 个样本的第 i 个特征（如第 3 套房面积 200）
- **mi** — 所有样本在特征 i 上的均值（(120+80+200)/3 ≈ 133）
- **σi** — 所有样本在特征 i 上的标准差（≈ 49）
归一化效果：
- 每个特征均值=0，标准差=1
- 所有特征尺度一致
- 梯度更新直指最低点，训练更快

## 代码示例（逐行解析）

```python
import torch
```
导入 PyTorch 库，类似 C 的 `#include`。

```python
x = torch.tensor([[120.0, 3.0], 
                  [80.0,  2.0], 
                  [200.0, 5.0]])
```
创建一个 3×2 的矩阵（张量），3 个样本，每个样本 2 个特征（面积、房间数）。

```
形式像 C 的二维数组：
  x[0] = [120.0, 3.0]    ← 样本1：面积120，房间3
  x[1] = [80.0,  2.0]    ← 样本2：面积80， 房间2
  x[2] = [200.0, 5.0]    ← 样本3：面积200，房间5
```

```python
mean = x.mean(dim=0)
```
计算每个特征的均值（dim=0 表示按列算）。

```
面积均值 = (120 + 80 + 200) / 3 = 133.3
房间均值 = (3 + 2 + 5) / 3 = 3.33
```

```python
std = x.std(dim=0)
```
计算每个特征的标准差。

```python
x_norm = (x - mean) / std
```
归一化公式：先减均值，再除标准差。

```python
print(x_norm)
```
输出归一化后的结果：

```
[[-0.27, -0.27],    ← 样本1归一化后
 [-1.08, -1.07],    ← 样本2归一化后
 [ 1.35,  1.33]]    ← 样本3归一化后
所有数值都在 0 附近了，两个特征的尺度一致。
```

🔗 **关联知识：**
- 跟 RMSProp/Adam 解决的是同一个问题，但 Normalization 在数据层面改，RMSProp 在优化器层面改
归一化（数据层面）：喂数据之前，把输入特征的尺度拉齐
RMSProp/Adam（优化器层面）：训练过程中，对不同参数用不同的有效步长
- 两者配合使用效果最好
